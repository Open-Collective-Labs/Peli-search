# v1 — Full-Text Search Foundation

Focus: Single-node full-text search with filtering, sorting, and aggregations.

## Features

### Index Management
- Create and delete indexes
- Define field mappings (text, keyword, integer, float, boolean)
- Configurable analyzers (standard, simple, whitespace, english)
- List and inspect indexes

### Document Operations
- Index single documents
- Bulk index (batch)
- Get document by ID
- Delete document by ID
- Automatic ID generation

### Search
- Match query (single field, multi-field)
- Phrase query (with slop)
- Boolean query (must, should, filter, must_not)
- Term query (exact match)
- Range query (numeric)
- Match all query

### Filtering
- Term filter
- Terms filter
- Range filter
- Exists filter
- Boolean filter composition (and/or)

### Sorting
- Ascending / Descending by field
- Multi-field sorting
- Sort by relevance or document ID
- Missing value handling

### Aggregations
- Terms aggregation (group by field value)
- Metrics (min, max, sum, avg, count, stats)
- Histogram aggregation
- Nested aggregations

### Storage
- Write-ahead log (configurable flush interval)
- Immutable segments (memory-mapped)
- Tiered merge policy
- Background compaction
- Checksum verification

### APIs
- REST API (Actix-web HTTP server)
- Embedded Rust SDK (library)

### Performance Targets
- Index throughput: 10K docs/sec (1KB each)
- Query latency: < 10ms (P50), < 50ms (P99)
- Index size: up to 10M documents per node

## Out of Scope for v1

- Vector search
- Hybrid search
- Clustering / replication
- Distributed search
- Query planner optimizations
- Authentication / authorization
