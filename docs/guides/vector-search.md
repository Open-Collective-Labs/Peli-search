# Vector Search

> **Phase 2 feature** — not yet available in v1.

Vector search enables similarity-based retrieval using embeddings. Documents are matched by semantic similarity rather than keyword overlap.

## Embeddings

Documents must include vector embeddings generated by a separate embedding model:

```json
{
  "id": "doc_1",
  "title": "Deep Learning with Transformers",
  "description": "A comprehensive guide to transformer architectures",
  "title_vector": [0.021, -0.043, 0.156, ...],
  "description_vector": [0.033, -0.021, 0.112, ...]
}
```

## Vector Fields

Define vector fields in the index mappings:

```json
{
  "mappings": {
    "properties": {
      "title_vector": {
        "type": "vector",
        "dimensions": 384,
        "similarity": "cosine"
      }
    }
  }
}
```

### Parameters

| Parameter | Description |
|-----------|-------------|
| `dimensions` | Vector dimensionality (must match the embedding model output) |
| `similarity` | Distance metric: `cosine`, `dot_product`, or `euclidean` |

## Similarity Search

```bash
curl -X POST http://127.0.0.1:7700/indexes/articles/search \
  -H "Content-Type: application/json" \
  -d '{
    "query": {
      "vector": {
        "field": "title_vector",
        "vector": [0.021, -0.043, 0.156, ...],
        "k": 10
      }
    }
  }'
```

### HNSW Index

Vector search uses a Hierarchical Navigable Small World (HNSW) index for approximate nearest neighbor search:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `m` | 16 | Number of connections per layer |
| `ef_construction` | 200 | Quality of index construction |
| `ef_search` | 100 | Search effort |
